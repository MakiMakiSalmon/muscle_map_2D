## A. 疲労モデルの正しさ

- **V-A1** 過去日時ワークアウトの減衰適用
  - `performedAt` 起点で疲労デルタを減衰させてから現在値に加算する
  - 「昨晩のトレを翌朝記録」しても疲労値が実態と一致する
- **V-A2** スライダー初期値を減衰後の現在値に変更
  - v2.0 機能要件 C2（保存済み値で初期化）の再定義。表示値とスライダー位置が常に一致する
- **V-A3** 疲労インパクトのセッション永続化
  - 保存時に計算した `fatigueImpacts`（減衰前の生デルタ）をセッションに保存。計算式を変更しても過去履歴の表示が変わらない
- **V-A4** RPE（主観的強度 1〜10・任意入力）による強度反映
  - `intensityFactor = 0.6 + RPE × 0.06`（未入力 = 1.0 で現行互換）
- **V-A5** `head` のラベルを「頭部」→「首」に変更（MuscleId `head` は不変）

## B. UX

- **V-B1** モバイルファーストレスポンシブ対応
  - 375px 幅で全機能が操作可能（体図上部・パネル縦積み・全画面モーダル・ボトムバー横スクロール）
  - md 以上は現行 2 カラム維持。Modal の focus trap 実装。**PWA は対象外（v3.1 以降）**
  - ログインに `signInWithRedirect` フォールバック（ポップアップブロック対策）
- **V-B2** エラーフィードバック基盤
  - Toast 通知（自作・パッケージ追加なし）。保存/リセット/記録の失敗を可視化
  - 疲労データ取得失敗時のエラー表示 + 再試行、401 時の再ログイン導線
- **V-B3** 履歴チャートの時間軸化
  - 実時間軸（直近 7 日）の折れ線 + 現在から完全回復までの予測破線
- **V-B4** ワークアウト保存の楽観的更新（v2.0 設計 §16-2 の実施）
- **V-B5** 「今日のおすすめ」表示（新機能）
  - 回復済み（グループ値 < 30%）の筋肉グループをダッシュボード上部に提示
- **V-B6** ワークアウトの削除（新機能・削除のみ）
  - セッション削除 + 由来スナップショット削除 + 現在値の再導出。**編集は対象外（削除 + 再入力で代替、v3.1 以降）**

## C. データ・性能・セキュリティ

- **V-C1** 現在疲労値の単一ドキュメント化
  - `users/{uid}/state/fatigueCurrent`（16 筋肉 map の 1 ドキュメント）。現在値取得が 16 reads → 1 read
  - 本番データなし（Q2）のためマイグレーションなし。欠落時はデフォルト値
- **V-C2** Firestore セキュリティルール強化
  - `users/**` へのクライアント直接アクセスを全遮断（全操作は API 経由のため）
- **V-C3** 種目カタログの拡充と取得仕様の整理
  - シードを 30〜40 種目へ拡充。全件取得モード（上限 200 件を仕様化）で名前解決の欠落バグを解消

## D. 品質・運用

- **V-D1** CI の push 対象を `[main, ver3.0]` に修正（統合ブランチの品質ゲート回復）
- **V-D2** API DTO 型の分離（`src/types/api.ts`。`recordedAt: Date/string` の型不整合を解消）
- **V-D3** Playwright E2E スモークテスト（ログイン→保存→色変化→記録→履歴の 1 本道）
- **V-D4** ドキュメント v3.0 化（README / CLAUDE.md / AGENTS.md 更新、v2.0 設計書 3 点は凍結注記のみ）
- **V-P0** バグ修正
  - ワークアウト実施日時初期値の UTC ズレ（JST −9h）修正

## リリース方針（確定）

- **一括リリース**: 中間リリースなし。全マイルストーン完了後に `ver3.0 → main` を単一 PR でマージ（Q1）
- Vercel の本番デプロイ対象は **main のみ**（Q9）。開発中の ver3.0 マージは本番に影響しない
- 本番 Firebase に実データなし（Q2）。全変更でデータマイグレーション不要
- Firestore rules / indexes は main マージ時の deploy.yml で本番反映

## 対象外（v3.1 以降で再検討）

- PWA / オフライン対応・ホーム画面追加（Q7）
- ワークアウトの編集（PATCH）（Q5。削除 + 再入力で代替）
- 回復完了時のブラウザ通知（v2.0 から継続除外）
- メール/パスワード認証（同上）
- Firestore → 他 DB への移行（同上）
- 体重ベースの強度係数（RPE 方式を採用したため）
- 多言語化
