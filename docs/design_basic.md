## 6. Firestoreデータ構造（概要）

詳細なフィールド定義は詳細設計書で定める。

```
Firestore
│
├── exercises/                    # 種目カタログ（グローバル、読み取り専用）
│   └── {exerciseId}
│       ├── nameJa: string
│       ├── primaryMuscles: string[]
│       └── secondaryMuscles: string[]
│
└── users/
    └── {uid}/
        ├── fatigueSnapshots/     # 疲労値履歴（保存のたびに追記）
        │   └── {snapshotId}
        │       ├── muscleId: string
        │       ├── value: number (0-100)
        │       ├── recordedAt: timestamp
        │       └── source: 'manual' | 'workout'
        │
        └── workoutSessions/      # トレーニングセッション
            └── {sessionId}
                ├── performedAt: timestamp
                ├── notes: string
                └── exercises: array  # 種目・セット情報
```

## 7. 技術的な主要方針

| 項目 | 方針 |
|------|------|
| 認証ガード | `(dashboard)` レイアウトでFirebase Auth状態を確認。未認証なら `/login` へリダイレクト |
| APIルート認証 | 全APIルートでFirebase IDトークンをBearerヘッダーで検証 |
| 回復計算 | APIレスポンス時にアプリ層で計算（`value × (1 - 経過時間h / 回復時間h)`）。バックグラウンドジョブ不要 |
| 楽観的更新 | TanStack Queryの`onMutate`でUI即時更新、エラー時に自動ロールバック |
| 状態分離 | サーバー状態 → TanStack Query、UIのみの状態（選択筋肉・タブ）→ Zustand |
| テスト | 回復計算などの純粋関数はUnitテスト必須。コンポーネントは主要インタラクションのみ |
| リージョン | Vercel `hnd1`（東京）+ Firestore `asia-northeast1`（東京）。日本ユーザー前提で同一リージョン配置。詳細設計 §19-1 参照 |
