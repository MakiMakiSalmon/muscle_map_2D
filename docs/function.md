# 機能一覧（v2.0 実装対象・確定版）

> **【凍結・歴史的記録】** 本書は v2.0 の確定仕様。v3.0 の生きた仕様は [docs/v3/](./v3/)（`function.md` / `design.md`）を参照。本書は v3.0 の design.md で上書きされていない箇所のみ有効。

筋肉疲労マップ 2D v2.0 の実装対象機能。スコープは確定済み。
詳細な実装仕様は [design_basic.md](./design_basic.md) / [design_detail.md](./design_detail.md) を参照。

---

## A. ユーザー管理

- **A2** ソーシャルログイン（Google Sign-in）
  - Googleアカウントでのワンクリックログイン
  - Firebase Auth を利用

## B. 疲労マップ表示

- **B1** 2D体図（前面・背面）の SVG 表示
  - 人体の筋肉部位を SVG で可視化。前面・背面の2ビュー
- **B2** 疲労値に応じた色分け表示（緑/黄/オレンジ/赤）
  - 0-29%→緑、30-59%→黄、60-79%→橙、80-100%→赤
- **B3** 回復進捗をリアルタイムで色に反映
  - 保存時刻からの経過時間に応じて疲労色を自動更新（E1 とセット）

## C. 疲労値入力

- **C1** 部位クリック → スライダーで手動入力（0-100%）
  - SVG の筋肉部位をクリックしてスライダーで疲労値を設定
  - 保存は明示的な `[確定]` ボタン押下（debounce 自動保存はしない）
- **C2** スライダー初期値を保存済みの値に設定
  - クリック時に現在の保存値をスライダーの初期位置として表示（旧実装では常に 50 固定だったバグの修正）
- **C3** トレーニング種目入力 → 疲労値の自動計算・加算
  - 「ベンチプレス 3セット」などを入力すると、関連筋肉に自動で疲労値を加算
  - 種目カタログ（`exercises` コレクション）を使用
- **C4** すべてリセット機能
  - 全筋肉の疲労値を 0 にリセット
  - インライン確認モーダル（M02）で誤操作を防止

## D. 履歴・分析

- **D1** 疲労値の永続化された時系列履歴チャート
  - 各筋肉の疲労値履歴を Firestore に保存し、ページ再読み込み後も参照可能
- **D2** 回復タイムライン表示（「あと Xh で回復」表示）
  - 各筋肉について完全回復までの残り時間を UI 表示
- **D3** 過去のトレーニングセッション一覧
  - 記録したトレーニング内容を日時順に閲覧（S03 画面）

## E. 自動回復システム

- **E1** 時間経過による疲労値の自動減衰
  - 最終保存時刻からの経過時間と筋肉ごとの回復率に基づき、現在の疲労値を自動計算
  - 回復率の目安:
    - 胸・肩・上腕二頭筋・上腕三頭筋: 約 48時間
    - 背中・太もも: 約 72時間
    - ふくらはぎ: 約 48時間
    - 前腕: 約 36時間
    - 頭部: 約 24時間

## F. インフラ・開発品質

- **F1** テストスイート（Vitest + React Testing Library + MSW）
  - ユニットテスト・コンポーネントテスト・APIテストの基盤を構築
- **F2** CI/CD（GitHub Actions）
  - push 時に自動で lint・test・build を実行
- **F3** デプロイ先: Vercel
  - GitHub リポジトリ連携で push 時に自動ビルド＆デプロイ

## G. 横断的な技術変更

- **G2** 状態管理を TanStack Query + Zustand に移行
  - サーバー状態は TanStack Query、UI 状態は Zustand
  - 旧実装の素の useState/useEffect によるロジック集中を解消

## 対象外（v1.3 以降で再検討）

- **A1** メール/パスワード認証（A2 に一本化）
- **E2** 回復完了時のブラウザ通知
- **G1** Firestore → Supabase への DB 移行（Firestore 継続・スキーマ再設計でカバー）
- ワークアウトセッションの編集・削除 API（MVP では手動スライダー上書き or 全リセットで代替）
- 強度（重量）を反映した疲労計算（MVP は volume のみ。詳細設計 §4-2 参照）
